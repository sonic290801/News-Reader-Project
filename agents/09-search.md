# Agent: Search

## Responsibility
Implement full-text search across all stored content (titles, article text, transcripts), with filters and saved keyword alerts.

## Output
- `app/(app)/search/page.tsx` — search UI
- `components/search/SearchBar.tsx` — global search input (in nav)
- `app/api/search/route.ts` — search endpoint
- `lib/search/index.ts` — search logic
- `app/api/alerts/route.ts` — saved search alerts CRUD
- DB migration: add `Alert` model + full-text search indexes

## Search Approach
PostgreSQL full-text search (no external dependency):
- Use `to_tsvector` on `title || ' ' || coalesce(excerpt,'') || ' ' || coalesce(fullText,'') || ' ' || coalesce(transcript,'')`
- Add a generated tsvector column to ContentItem for performance
- Query with `to_tsquery` or `websearch_to_tsquery` (handles natural language queries)
- Rank results with `ts_rank`

Prisma raw queries via `prisma.$queryRaw` for the full-text parts; standard Prisma filters for the filter combos.

## DB Changes

Add to ContentItem in schema.prisma:
```prisma
searchVector  Unsupported("tsvector")? // generated by DB trigger

@@index([searchVector], type: Gin) // GIN index for fast FTS
```

Migration SQL (run alongside prisma migrate):
```sql
-- Add generated search vector column
ALTER TABLE "ContentItem" ADD COLUMN "searchVector" tsvector
  GENERATED ALWAYS AS (
    to_tsvector('english',
      coalesce(title, '') || ' ' ||
      coalesce(excerpt, '') || ' ' ||
      coalesce("fullText", '') || ' ' ||
      coalesce(transcript, '')
    )
  ) STORED;

CREATE INDEX "ContentItem_searchVector_idx" ON "ContentItem" USING GIN ("searchVector");
```

## API Route — app/api/search/route.ts
GET with params:
- `q` — search query (required, min 2 chars)
- `sourceId`, `category`, `type`, `from`, `to` — optional filters
- `page`, `limit` — pagination (default limit 30)

Returns: `{ results: SearchResult[], total: number, hasMore: boolean }`

Each SearchResult: `{ id, title, sourceLabel, type, publishedAt, snippet, url }`

## Search Page — app/(app)/search/page.tsx
- Search input (pre-filled from URL param `?q=`)
- Filter bar: same source/category/type/date filters as feed
- Results list: similar to List View rows but with highlighted snippet instead of excerpt
- "No results" state with suggested related terms
- Result count: "23 results for 'Gaza ceasefire'"
- Clicking a result: opens Reader View

## Global Search Bar — components/search/SearchBar.tsx
- Appears in the nav shell (top of page on mobile, top of sidebar on desktop)
- Keyboard shortcut: Ctrl+K / Cmd+K to focus
- Type-ahead: after 400ms debounce, shows top 5 results in a dropdown
- "See all results" → navigates to /search?q=...
- Press Escape to close dropdown

## Alerts Model

```prisma
model Alert {
  id        String   @id @default(cuid())
  keyword   String
  lastMatch DateTime?
  createdAt DateTime @default(now())
}
```

## API Routes — app/api/alerts/route.ts
- GET → list all alerts
- POST `{ keyword: string }` → create alert
- DELETE `{ id: string }` → remove alert

## Alerts UI
Simple list in Settings page (not a separate page):
- Input + "Add alert" button
- List of saved keywords with delete buttons
- Each keyword shows "Last matched: X ago" or "No matches yet"

## Acceptance Criteria
- Searching "Ukraine" returns articles and Reddit posts containing that word, ranked by relevance
- Searching within a YouTube transcript returns that episode
- Highlighted snippet shows the matched term in context (bolded)
- Filters narrow results correctly
- Ctrl+K focuses the search bar from anywhere in the app
- Creating an alert for "ceasefire" and ingesting a matching article marks the alert as matched
- Empty query returns an appropriate message, not an error
